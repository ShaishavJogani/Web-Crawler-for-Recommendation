You may need reference class JAR file within JAR file . For example , typical situation applet bundle JAR file whose manifest reference different JAR file ( several different JAR file ) serve utility purpose applet . You specify class include Class-Path header field manifest file applet application . The Class-Path header take following form : Class-Path : jar1-name jar2-name directory-name/jar3-name By use Class-Path header manifest , avoid specify long -classpath flag invoke Java run application . Note : The Class-Path header point class JAR file local network , JAR file within JAR file class accessible Internet protocol . To load class JAR file within JAR file class path , must write custom code load class . For example , MyJar.jar contain another JAR file call MyUtils.jar , use Class-Path header MyJar.jar 's manifest load class MyUtils.jar class path . An Example We want load class MyUtils.jar class path use MyJar.jar . These two JAR file directory . We first create text file name Manifest.txt following content : Class-Path : MyUtils.jar Warning : The text file must end new line carriage return . The last line parse properly end new line carriage return . We create JAR file name MyJar.jar enter following command : jar cfm MyJar.jar Manifest.txt MyPackage/*.class This create JAR file manifest following content : Manifest-Version : 1.0 Class-Path : MyUtils.jar Created-By : 1.7.0_06 ( Oracle Corporation ) The class MyUtils.jar load class path run MyJar.jar .
