If application bundle JAR file , need way indicate class within JAR file application 's entry point . You provide information Main-Class header manifest , general form : Main-Class : classname The value classname name class application 's entry point . Recall entry point class method signature public static void main ( String [ ] args ) . After set Main-Class header manifest , run JAR file use following form java command : java -jar JAR-name The main method class specify Main-Class header execute . An Example We want execute main method class MyClass package MyPackage run JAR file . We first create text file name Manifest.txt following content : Main-Class : MyPackage.MyClass Warning : The text file must end new line carriage return . The last line parse properly end new line carriage return . We create JAR file name MyJar.jar enter following command : jar cfm MyJar.jar Manifest.txt MyPackage/*.class This create JAR file manifest following content : Manifest-Version : 1.0 Created-By : 1.7.0_06 ( Oracle Corporation ) Main-Class : MyPackage.MyClass When run JAR file following command , main method MyClass executes : java -jar MyJar.jar Setting Entry Point JAR Tool The 'e ' flag ( 'entrypoint ' ) create override manifest 's Main-Class attribute . It use create update JAR file . Use specify application entry point without edit create manifest file . For example , command create app.jar Main-Class attribute value manifest set MyApp : jar cfe app.jar MyApp MyApp.class You directly invoke application run following command : java -jar app.jar If entrypoint class name package may use ' . ' ( dot ) character delimiter . For example , Main.class package call foo entry point specify following way : jar cfe Main.jar foo.Main foo/Main.class
