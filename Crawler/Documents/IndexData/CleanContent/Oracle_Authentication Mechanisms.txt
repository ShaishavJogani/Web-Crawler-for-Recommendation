Different version LDAP support different type authentication . The LDAP v2 defines three type authentication : anonymous , simple ( clear-text password ) , Kerberos v4 . The LDAP v3 support anonymous , simple , SASL authentication . SASL Simple Authentication Security Layer ( RFC 2222 ) . It specify challenge-response protocol data exchange client server purpose authentication establishment security layer carry subsequent communication . By use SASL , LDAP support type authentication agree upon LDAP client server . This lesson contain description authenticate use Anonymous , Simple , SASL authentication . Specifying Authentication Mechanism The authentication mechanism specify use Context.SECURITY_AUTHENTICATION environment property . The property may one following value . Property Name Property Value sasl_mech A space-separated list SASL mechanism name . Use one SASL mechanism list ( e.g. , `` CRAM-MD5 '' mean use CRAM-MD5 SASL mechanism describe RFC 2195 ) . none Use authentication ( anonymous ) simple Use weak authentication ( clear-text password ) The Default Mechanism If client specify authentication environment property , default authentication mechanism `` none '' . The client treat anonymous client . If client specify authentication information without explicitly specify Context.SECURITY_AUTHENTICATION property , default authentication mechanism `` simple '' .
